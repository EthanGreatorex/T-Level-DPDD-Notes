Data

How is data generated?

  • Human generated
  • Artificial intelligence/machine learning
  • Sensors
  • Transactional data

Knowledge

Knowledge is a combination of information and a person’s experience and intuition

Information

This is data which has been organised and given analysis and context

Data

This is the raw numbers and figures which lack any kind of organisation

Why do organisations need data and information?

  • Companies that do not understand the importance of data management are less likely to survive
  • A company’s data is its most valuable asset
  • It gives valuable insights to customer needs and user behaviours

Directory-based structures

  • Often in the format of a hierarchical tree structure
  • Easier for the end user to locate files
  • Should be a logical structure

Data wrangling

  • Process of changing unorganised and raw data into data that can be analysed, therefore making it useful.

Six steps in data wrangling

  • Discovery
    • Understand the data fully
  • Structure
    • Data must be structured to ensure it is accessible
  • Cleansing
    • Data sets can include data that does not meet the requirements, causing a skew in the analysis
  • Enrichment
    • Externally collected data sets will most likely not meet the specified needs.
    • Two or more data sets will need to be combined
  • Validation
    • Checking the reliability, quality and safety of the data.
    • This involved checking the data to ensure it is complete
  • Publishing
    • When the data has been wrangled, the data is ready to be analysed and produce output
    • e.g., textual or graphical

Data systems

All data systems have the same basic functions

  • Input
    • Collection of raw data
  • Search
    • Carry out searches on data
  • Save
    • Data that has been input into the system must be stored for reuse
  • Integrate
    • Different data types/formats can be integrated into a single location
  • Organise (index)
    • Saved data can be organised, e.g., through indexing
  • Output
    • The processed and analysed data can be output to the relevant people
  • Feedback loop
    • Output that is returned to, usually senior management, to evaluate various processes

Data entry and maintenance

There are two main types of errors that can be made on data entry

  • Transcription
    • Occurs when copying data from the source e.g., from a user.
    • e.g., the user may type the wrong key on the keyboard
  • Transposition
    • Occurs when two letters or numbers have been reversed

Validation

Validation: Checks the data being entered is reasonable and meets pre-set rules

Verification: Checks if the data entered is identical to the source.

ERD

An entity relationship diagram is used to define how a system will be implemented, this is done through the use of entity's and relationships.

Primary key: A field in a table that uniquely identifies each record

Foreign key: Used to link tables together, a foreign key references a primary key in a different table

Data dictionary

  • Contains data about the data that will be included in the database
  • → metadata

Six Vs

  • Volume
    • The amount of data generated each second, minute and hour
  • Variety
    • Considers where the data has come from and in what form
  • Velocity
    • The speed at which the data is being generated in real time
  • Variability
    • How quickly and to what extent, the data being processed is changing
  • Value
    • The meaning of the data after processing and analysis
    • How much the results meet the goals
  • Veracity
    • How trustworthy is the data

Data assurance: The data must have a high level of reliability and quality.

Data redundancy: Occurs when the same piece of data is stored two or more times on a system

Qualitative: Text/Descriptive based data

Quantitative: Numerical based data

Ethics

  • All data should be gathered, entered and maintained in the same way with no bias
  • Data must be kept securely

Big data: Very large data set that can be analysed to produce information

Data warehousing & lakes

  • Data warehouse
    • Central collection of key data, in a pre-defined format.
    • Used to make business-critical and evidence-based decisions
    • Subject to data cleaning
  • Data lake
    • Hold data in an unstructured way+
    • Stored in raw state
    • It may not be processed
    • Takes from all sources in any data type

Metadata

  • Data about data
  • Can be classified in three main ways:
    1. Administrative
    2. Descriptive
    3. Structural

1 → Provides administrative instructions about a file, such as any restrictions to be placed on the file.

2 → Relates to how the data is identified, could include titles, dates or keywords. (referred to as

metatags on a website)

3 → Structural describes how a digital asset is arranged or structured, an obvious example is a data dictionary

API

  • Application Programming Interface
  • Enables users to manage, access and use data across a range of platforms
  • Main advantage is users do not need to create accounts for multiple websites
  • Users can create many websites using an API whilst only requiring one log-in credential
  • Uses JSON to send and receive requests
  • API sends a user’s request to a digital system, and the system sends back a response
  • e.g., a user adds a product to their shopping basket. The API will tell the website that a product has been added to the basket and the website puts the product into the user’s virtual basket. The basket is then updated.

Data in movement

Data at rest

  • Data that is stored and not moving

Data in use

  • Data that is being actively used and/or processed by a user or digital system

Data in motion

  • Data when moving from one location to another, this includes moving inside a network
  • Data in motion is less secure than the other two
