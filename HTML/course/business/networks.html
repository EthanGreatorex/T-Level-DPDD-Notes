Networks

Why do business use networks?

Businesses use networks to connect together digital devices, wirelessly or with cables, so that they can communicate and share resources with each other

Bandwidth & Latency, and Their Effect on Performance

Bandwidth

  • Measurement of how much data can be transferred from one point to another during a specific time period
  • Larger the available bandwidth the greater amount of data can be sent and received across the network

Latency

  • Measure of the time taken to transmit data from one point to another

Factors that Impact the Latency of a Network

  • Distance → The response time will be longer based on the distance between the client and server e.g., another country
  • Transmission medium → Wired has lower latency than wireless, radio signals degrade more over time
  • Propagation delay → The length of time it takes data packets to travel from the device to the server

Different Types of Network Models

Type Description
Client-server The clients and server communicate with each other through the local network or the internet. The server is used to provide resources to the client.
Thin client Runs by accessing resources on a central server. They do not have local storage and have a ‘thinned’ version of the operating system.
Peer-to-Peer Each compute is a ‘peer’ to the other computer. The ‘peer’ acts as a client and server.

Type Description Advantages Disadvantages
Star Each node is connected to a central device. The central device receives signals from anode and transmits it to the other connected nodes on the network. Easy to install

No disruption to the network when connecting or removing nodes

Easy to troubleshoot; all data goes through a central point
If the hub fails, the network goes down

It can be costly because of the hub or switch
Mesh Each device has a point-to-point contact with every other device on the network. No traffic problems occur as there are dedicated links

If one link fails, it does not impact the whole system

Data travels down a single line and is therefore more secure
A lot of wiring involved and is difficult to maintain

As there is a connection between each node, installation is complex and costly
Tree Also known as star bus topology. Groups of star configured nodes are connected to a linear bus backbone cable. Other nodes on the network are not affected if one node fails

It supports the expansion of existing networks

Provides a hierarchical arrangement of the nodes

The bigger the network, the more complicated it becomes to configure.

Performance can be slow due to the number of nodes on the network.

Latency is decreased because of the smaller network size

Easier to troubleshoot problems on a smaller network than a large one

Application

  • Closest layer to the user
  • Uses protocols such as → FTP, HTTP,

Presentation

  • Translates data into a suitable format so that it can be understood by the application
  • Uses protocols such as → JPEG, MPEG

Session

  • This allow applications running on different computers to communicate with each other.
  • E.g., Establish session, manage data transfer, tear down the session
  • Uses protocols such as → NFS, SQL

Transport

  • Responsible for ensuring reliable data delivery so that packets arrive error free and without loss.
  • Uses protocols such as → TCP, UDP (User Datagram Protocol)

Network

  • This layer is responsible for moving data around networks e.g., the internet

  • This will route packets across a static or dynamic route through routers until the destination is reached

  • This layer:

    • Detect transmission errors (unless UDP is used)
    • Regulate the speed of data transfer
    • Break down packets into smaller chunks if required

  • Uses protocols such as → IP

Data Link

  • This layer is responsible for transferring data between devices
  • It is responsible for:
    • Encodes bits into packets prior to submission and decodes packets into bits at the destination
    • Logical link control, media access control and hardware addressing

Physical

  • This is responsible for the transmission of data over network communication media
  • This layer includes:
    • Network medium
    • Physical network topologies
    • Network Interface Card

TCP/IP Four-Layer Model

Layer Description
Application This contains a suite of protocols which encode/decode messages into a form that is understood by the application
Transport The data from the application layer is broken down and formatted into segments. Each segment is given a header with contains a sequence number so that TCP can keep track of each segment. A checksum is calculated and added to the header to help with error detection
Internet This takes the segments from the transport layer and encapsulates them into IP packets. Each IP packet is given a header containing data such as the source IP address and the destination IP address.
Network interface This enabled the physical transfer of packets. Once a link has been chosen e.g., Ethernet, Wi-Fi, Bluetooth, the appropriate link layer protocol is selected

Role of Data Packets in Transmitting Over A Network

Contiguous data :: data that is stored in a collection of adjacent locations

Time to live (TTL) :: _the amount of time or 'hops' that a packet is set to exist inside a network before being discarded by the router

Hops :: the number of routers that a packet passes through from its source to its destination

Data sent over a network is referred to as packets. Large packets of data are broken down into smaller datagrams. When the packets reach their final destination, the datagrams are reassembled a single file or block of contiguous data.

What do data packets consist of?

Data packets consist of a header, payload and trailer

Header
  • Contains instruction about the data
  • IP address of the sender of the packet
  • Destination IP address of the receiver of the packet
  • Internet protocol
  • Size of the header and payload
  • Number of hops
  • Time to Live
  • Checksum
  • Packet number (used for sequencing)
Payload

This is the body of the data packet and contains the actual data that is being sent/received

Trailer

This is sometimes referred to as the footer and is used to inform the receiving device that it is the end of the packet. The trailer also includes error checking.
