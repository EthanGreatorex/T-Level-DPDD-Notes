Security

External stakeholders: groups outside an organisation, for example shareholders

Internal stakeholders: groups within an organisation, for example owners and employees

What Sort of Information Do Organisations Store?

  • Employee salaries
  • Employee perks
  • Client lists
  • Trade secrets
  • Sales numbers
  • Customer information

What are the Impacts of Failing to Maintain Privacy and Confidentiality?

Financial

  • Possible payment of compensation
  • Increased costs to improve security and new hardware
  • Loss of customers leading to loss of revenue

Reputation

  • Negatively affected reputation
  • No longer seen as trustworthy

Downtime

  • Loss of time when restoring loss or corrupted data
  • Loss of time is the service is disrupted

Potential Technical Threats and Vulnerabilities to Systems, Data and Information

Cyber security: the practise of defending computers, servers, mobile devices, electronic systems, networks and data from malicious attacks

Man in the Middle Attacks

  • A hacker places themselves in the middle of a communication between two devices
  • The hacker can intercept, send and receive data for someone else

CIA Triad

CIA: confidentiality, integrity and availability

Confidentiality

  • The digital systems, data and information resources are protected from unauthorised access

Integrity

  • Data is protected from unauthorised changes to ensure it is reliable and correct

Availability

  • Authorised users have access to the systems, data and information they require

Processes and Procedures to Mitigate Threats

Air gapping

  • A digital system that is physically isolated from potentially dangerous networks, such as the internet
  • A digital system that works offline

Anti-virus

  • Software that is designed to prevent, detect and remove viruses and other malware
  • Usually will either quarantine a file or delete it

APIs

  • APIs enables two or more software applications to interact
  • Typically can call into three different levels of certification
Private
  • API is only used internally within the business; the most secure certification
Partner
  • API is only available to trusted partners of the business or organisation
Public
  • API is available to everyone

Encryption

Data encryption software: software that is used to encrypt a file or data

Encryption code/key: a set of characters, a phrase or numbers that are used when encrypting or decrypting data or a file

Asymmetric Encryption

  • Uses a public/private key pair
  • The encryption key is made available to anyone to use and encrypt data, but only the person who receives the data receives the decryption key (private).

Symmetric Encryption

  • The same key is used for both encryption and decryption
  • Same key is shared between the sender and the recipient

Hashing

Hash: a number generated from a string of text

  • Uses an algorithm to map data of any size to a fixed length
  • This is called a hash value
  • Hashing verifies that a file or piece of data is authentic and not been altered
  • Hashing is only one way and is not reversible (without tremendous power)
  • Example; user enters a password --> password is hashed --> hashed entry is compared with hashed real password --> result of comparison is returned.

Multi-factor Authentication

  • Also know as 2FA

Possession Based

  • Based on a possession the user has with them at all times e.g., a phone

Biometric

  • Uses a person's physical characteristics to authenticate

Knowledge

  • Authenticates by using questions and answers which have already been agreed

Location Based

  • Authenticates a user through their physical location

Firewalls

  • A security device that mitigate against threats by examining data packets

Software Firewall

  • A program that monitors traffic through port numbers and applications

Physical Firewall

  • A piece of hardware installed between the network and the gateway

Packet Filtering

  • Analyses data packets and blocks any packets that do not meet the predefined security rules

Security Testing

Black Box

  • The people carrying out the rests are provided with no information except the name of the business or organisation.
  • Carried out from an external perspective

White Box

  • The people carrying out the tests are provided with full and complete information about the digital system to be tested.
